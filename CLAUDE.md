# Supersonik GTM System — Master Playbook

This is the Claude entry point for the operational GTM system. v1 ships **outbound** (email + LinkedIn DMs). v2+ will reuse account data for nurture + ads.

The research files at the project root (`../00_index.md` … `../12_icp.md`) are the **research base** — read-only context. Everything generated by the system lives under `gtm/`.

---

## Where each thing lives

| Need | Folder |
|---|---|
| Provider API keys, integration docs | `integrations/` |
| Voice DNA for team members | `content/voices/` |
| Per-channel composition rules + hygiene | `content/platform_adaptation/` |
| Tier table (3 rows) | `icp/tiers.md` |

| Signals → score model (v2 stake) | `accounts/scoring/` |
| SQLite DB schema + init script | `db/` |
| User workflow docs | `workflows/` |
| Slash commands | `.claude/commands/` |

Each folder has its own `CLAUDE.md` with focused rules. Load only what the task needs.

## Hygiene + anti-slop rules

Hygiene rules (no fabricated logos / ARR / customer counts / pricing / dates; trace every Supersonik claim to a sourced research file) and the global anti-slop kill list (banned words, banned patterns) live in **`content/platform_adaptation/global.md`**.

Read that file before generating any outbound or content. Do not duplicate its rules elsewhere — link to it.

## Workflows

### #1 — Create Campaign

Slash command: `/create-campaign`
Full doc: `workflows/create_campaign.md`

Step order:
1. Choose campaign type → `campaigns/campaign_types.md`
2. Choose framework → `campaigns/frameworks/frameworks_list.md`
3. Choose sequence → `campaigns/sequences/templates/*.yml` (default: `email_3step_default.yml`)
4. Load lead list → CSV input or pull from `accounts/` + DB
5. Choose voice → `content/voices/*.md`
6. Review all selections
7. Generate messages → `campaigns/messages/_generation_flow.md`

### #2 — Signal alert (deferred to v2)

Out of scope until Slack integration lands.

## MD↔HTML convention

Every editable file uses **YAML frontmatter for structured fields + markdown body for prose**. HTML generators render forms from frontmatter, docs from body. When editing tabular files (tiers, archetype index, frameworks_list, active_campaigns), edit the frontmatter array — the body table can be rebuilt from it.

---

## Naming

All new files and folders are `lower_snake_case`. The voice/archetype filenames are `<identifier>_<topic>.md` (e.g. `dani_carmona_voice.md`, `a_alex_cro.md`).
